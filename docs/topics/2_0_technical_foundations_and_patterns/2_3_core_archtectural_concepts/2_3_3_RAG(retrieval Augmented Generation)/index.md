# Retrieval Augmented Generation

## Overview
Retrieval-Augmented Generation (RAG) systems are a foundational component in building context-aware and knowledge-grounded applications using Large Language Models (LLMs). This section focuses on the core building blocks of RAGâ€”including ingestion pipelines, retrieval pipelines, and generation workflows. It also covers best practices and implementation strategies for tuning RAG systems to optimize response quality and relevance across various enterprise use cases.

## Why It Matters
By the end of this section, you will understand:

- How to implement the RAG pipeline

- Different patterns in the advance RAG pattern

- How to improve the performance of the RAG pipeline

- Practical techniques for implementing the RAG pipeline, along with common challenges and strategies to overcome them

Understanding these topics is important for building an effective RAG implementation, which serves as a foundational building block for most enterprise use cases. It enables large language models to operate on proprietary enterprise data while significantly reducing hallucinations.

Prerequisites: Basic understanding of Gen Core concepts and familiarity with similarity search is helpful but not required.
